2. HTML Tags with Attributes

Headings · Semantic tags · Forms · Tables

TAG REFERENCE

Important HTML Tags

  • Heading Tags: <h1> to <h6> - h1 largest, h6 smallest
  • <q>: Short quotation - browsers auto-add quotes
  • <strong>: Bold with semantic meaning (use over <b>)
  • <small>: Smaller text for footnotes, disclaimers

  • <del>: Deleted text - shows strikethrough

Categories

  • Formatting: <b>, <i>, <u>, <strike>
  • Semantic: <strong>, <em>, <q>, <small>
  • Special: <sub>, <sup>, <ins>, <del>

4. XML (eXtensible Markup Language)

Structure · DTD · Comparison with HTML

KEY POINTS

XML Essentials

  • Self-descriptive markup - tags describe data
  • Focus on WHAT data IS (vs HTML focuses on HOW it looks)
  • Must have XML declaration: <?xml version="1.0"?>
  • Must have one root element
  • Tags are case-sensitive
  • Proper nesting required
  • DTD (Document Type Definition) can validate structure

7. Database Access Concepts

SQL · Prepared Statements · PDO

⚠️ CRITICAL

SQL Injection Prevention

Vulnerable Code: 
$id = $_GET["id"];
$result = $conn->query("SELECT * FROM students WHERE id = " . $id);
// User can input: 1 OR 1=1 (returns all records!)
Secure Code - Use Prepared Statements: 
$stmt = $conn->prepare("SELECT * FROM students WHERE id = ?");
$stmt->execute([$_GET["id"]]);
// Separates data from SQL code - prevents injection
PDO

PHP Data Objects (PDO)

  • Modern secure way to access databases
  • Works with multiple database types
  • Uses prepared statements by default
  • Provides exception handling

Basic SQL Commands

CREATE TABLE students (id INT, name VARCHAR(100));
INSERT INTO students VALUES (1, "John");
SELECT * FROM students WHERE age > 20;
UPDATE students SET age = 21 WHERE id = 1;
DELETE FROM students WHERE id = 1;

8. Sessions and Cookies

Client-side vs Server-side storage

COOKIES

Cookies

  • Stored on CLIENT browser
  • Sent with every HTTP request
  • Size limit: ~4KB per cookie
  • Not secure - user can view/modify
  • Use for: Preferences, tracking
// Create
setcookie("name", "value", time()+3600, "/");
// Access
$_COOKIE["name"];
SESSIONS

Sessions

  • Data stored on SERVER
  • Identified by Session ID in cookie
  • Large data can be stored
  • More secure - data never visible to user
  • Use for: Login, shopping cart
session_start(); // call first!
$_SESSION["username"] = "John";
session_destroy(); // end session
