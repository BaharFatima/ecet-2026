OOPs Concepts
Foundation of Object-Oriented Programming

OOP is a programming paradigm based on objects and classes, providing better code organization and reusability through encapsulation, inheritance, and polymorphism.

Four Pillars of OOP:
1. Encapsulation: Bundling data and functions into a class, hiding internal details from the outside world. Using private, protected, and public access specifiers.
2. Abstraction: Hiding complex implementation details and showing only essential features. Achieved through abstract classes and interfaces.
3. Inheritance: Creating new classes from existing classes, promoting code reuse. Child class inherits properties from parent class.
4. Polymorphism: Same function name with different behaviors. Two types: compile-time (function overloading) and runtime (virtual functions).

Key Points for MCQ:

• Encapsulation increases code security by restricting direct access

• Inheritance saves code through reusability

• Polymorphism enables flexible code design

• A class is a blueprint; an object is an instance

C++ Keywords
Essential keywords for OOP programming
Access Specifiers:
public: Members accessible everywhere
private: Members accessible only within the class
protected: Members accessible in derived classes
Important Keywords:
class: Defines a class
this: Pointer to current object
virtual: For polymorphism
friend: Grants access to private members
MCQ Tip: By default, class members are private. Struct members are public by default.
Classes and Objects
Building blocks of OOP
Basic Syntax:
class ClassName { private: int x; // Data member public: void display() { // Member function cout << x; } }; ClassName obj; // Object creation

Key Differences:

• Class: User-defined data type combining data and functions

• Object: Instance of a class created in memory

• Member variables: Data stored in each object

• Member functions: Methods that operate on member variables

Array of Objects
Multiple objects in a single array
class Student { int rollNo; string name; }; Student students[100]; // Array of 100 objects students[0].rollNo = 1; // Access using dot operator
MCQ Focus: When creating an array of objects, default constructor is called for each element. Use parameterized constructor during declaration or manually initialize each object.
Accessing object members: - Using object: obj.member - Using pointer: ptr->member or (*ptr).member
Passing and Returning Objects
Passing by Value:
void func(Student s) { // Copy of object passed s.display(); }
Passing by Reference:
void func(Student &s) { // Reference passed (no copy) s.display(); // More efficient }
Returning Objects:
Student getStudent() { Student s; return s; // Object copy returned }

• Passing by value creates a copy (slower, safe)

• Passing by reference is faster (no copy overhead)

• Return by reference is risky (don't return local objects)

Pointers to Objects
Dynamic memory management
class Student { int id; void display() {} }; Student *ptr; // Pointer declaration ptr = new Student(); // Dynamic allocation ptr->id = 10; // Access using arrow operator ptr->display(); // Call member function delete ptr; // Free memory

Key Points:

• Pointer stores memory address of an object

• new allocates memory on heap, returns address

• delete frees dynamically allocated memory

• Arrow operator (->) used with pointers

• Dot operator (.) used with objects

MCQ Tip: sizeof(pointer) = 4 or 8 bytes (depends on architecture), regardless of what it points to.
References
Aliases to existing variables
int x = 10; int &ref = x; // ref is alias for x ref = 20; // x is now 20 class Student { int id; }; Student obj; Student &objRef = obj; // Reference to object objRef.id = 5;

• Reference is alias, not a pointer

• Must be initialized at declaration

• Cannot be reassigned

• No NULL references (references must refer to valid variable)

• Reference size = size of variable it references

Exam Key Difference:

Pointer vs Reference: Pointer can be NULL, reassigned, and uses -> or (*ptr). operator. Reference is non-NULL alias, initialized once, uses dot operator like original variable.

this Pointer
Pointer to current object
class Student { int id; void setId(int id) { this->id = id; // this->id = parameter id } void display() { cout << this->id; } }; Student s; s.setId(5);

• this is implicit pointer to current object

• Available in non-static member functions

• Type: const ClassName *this

• Used to avoid ambiguity between member and local variables

• Can be used to return object: return *this;

Important: this is a const pointer (address cannot change), but can modify object data (data is not const).

• setw(): Set field width

• setprecision(): Set decimal places

• setfill(): Set fill character

• left/right: Alignment

• hex/oct: Number base

Constructors
Initializing objects automatically
Default Constructor:
class Student { int id; string name; Student() { // Default constructor id = 0; name = ""; } };
Parameterized Constructor:
class Student { int id; string name; Student(int i, string n) { // Constructor with parameters id = i; name = n; } }; Student s(5, "John"); // Calling parameterized constructor
Copy Constructor:
class Student { int id; Student() { id = 0; } Student(const Student &s) { // Copy constructor id = s.id; } }; Student s1(5); Student s2(s1); // Copy constructor called

• Constructor has same name as class

• No return type, not even void

• Called automatically when object created

• Default constructor created by compiler if not defined

• Copy constructor used to copy one object to another

MCQ Key: Compiler provides default constructor, copy constructor, and copy assignment operator automatically. You can override them.
Destructors
Cleaning up when object is destroyed
class Student { int *ptr; Student() { ptr = new int[100]; } ~Student() { // Destructor delete[] ptr; // Free memory } }; Student s; // Constructor called // Destructor called when s goes out of scope

• Name: ~ClassName()

• No parameters, no return type

• Called automatically when object destroyed

• Used to free dynamically allocated memory

• Only one destructor per class

• Called in reverse order of construction

Memory Management: Always use destructor to free memory allocated in constructor with new. Use delete[] for arrays, delete for single variables.

Function Overloading
Same function name, different parameters
class Calculator { int add(int a, int b) { return a + b; } double add(double a, double b) { // Different parameter types return a + b; } int add(int a, int b, int c) { // Different number of parameters return a + b + c; } }; Calculator c; cout << c.add(5, 10); // Calls first function cout << c.add(5.5, 10.5); // Calls second function cout << c.add(5, 10, 15); // Calls third function

• Same function name with different parameters

• Can differ in: number, type, or order of parameters

• Return type alone cannot differentiate overloaded functions

• Resolved at compile time (compile-time polymorphism)

• Also called static or early binding

MCQ Rule: Return type difference only = NOT overloading. Compiler error!
Constructor Overloading
Multiple constructors with different parameters
class Student { int id; string name; Student() { id = 0; name = ""; } Student(int i) { id = i; name = ""; } Student(int i, string n) { id = i; name = n; } Student(string n) { id = 0; name = n; } }; Student s1; // Default constructor Student s2(5); // Constructor with int Student s3(5, "John"); // Constructor with int and string Student s4("John"); // Constructor with string
Delegation Constructor (C++11): Constructor can call another constructor of same class using : Constructor1() syntax

• Multiple constructors with same name

• Different parameter lists

• Resolves which constructor to call based on arguments

• Default constructor has no parameters

Operator Overloading
Redefining operators for custom types
Unary Operator Overloading:
class Complex { int real, imag; Complex operator++() { // Pre-increment real++; imag++; return *this; } Complex operator++(int) { // Post-increment Complex temp = *this; real++; imag++; return temp; } Complex operator-() { // Unary minus return Complex(-real, -imag); } }; Complex c(3, 4); ++c; // Pre-increment c++; // Post-increment -c; // Unary minus
Binary Operator Overloading:
class Complex { int real, imag; Complex operator+(const Complex &c) { // Addition return Complex(real + c.real, imag + c.imag); } Complex operator-(const Complex &c) { // Subtraction return Complex(real - c.real, imag - c.imag); } bool operator==(const Complex &c) { // Equality return real == c.real && imag == c.imag; } }; Complex c1(3, 4), c2(1, 2); Complex c3 = c1 + c2; // Addition bool equal = (c1 == c2); // Equality
Friend Function for Operator Overloading:
class Complex { int real, imag; friend Complex operator*(const Complex &c1, const Complex &c2); }; Complex operator*(const Complex &c1, const Complex &c2) { return Complex(c1.real * c2.real, c1.imag * c2.imag); }

• operator keyword followed by operator symbol

• Cannot overload: . :: .* ?: sizeof

• Assignment (=), subscript ([]), function call (()), member access (->) must be member functions

• Most operators can be member or friend functions

• Stream operators (<<, >>) typically friend functions

Key Differences:

• Member function: lhs is implicit *this, rhs is parameter. Example: a + b calls a.operator+(b)

• Friend function: Both operands are parameters. Example: friend operator+(a, b)

Single Inheritance
One derived class from one base class
class Vehicle { // Base class int speed; void display() { cout << speed; } }; class Car : public Vehicle { // Derived class int wheels; void show() { display(); // Can call base class function cout << wheels; } }; Car c; c.display(); // Inherited function c.show(); // Own function

• Base class: Parent class providing features

• Derived class: Child class inheriting from base

• public inheritance: public members remain public

• protected inheritance: public becomes protected

• private inheritance: all become private

Multiple Inheritance
One derived class from multiple base classes
class Vehicle { void vehicleInfo() {} }; class Engine { void engineInfo() {} }; class Car : public Vehicle, public Engine { // Multiple inheritance void carInfo() {} }; Car c; c.vehicleInfo(); // From Vehicle c.engineInfo(); // From Engine c.carInfo(); // Own function
Diamond Problem: When a class inherits from two classes that both inherit from a common base class, the common members appear twice. Solution: Use virtual inheritance.
class A { }; class B : virtual public A { }; // Virtual inheritance class C : virtual public A { }; class D : public B, public C { }; // No duplication
Multilevel Inheritance
Chain of inheritance: A -> B -> C
class Animal { void eat() { cout << "Eating"; } }; class Mammal : public Animal { void walk() { cout << "Walking"; } }; class Dog : public Mammal { // Inherits from Mammal, indirectly from Animal void bark() { cout << "Barking"; } }; Dog d; d.eat(); // From Animal d.walk(); // From Mammal d.bark(); // Own function
Constructor Calling Order (Multilevel): Base class -> Intermediate class -> Derived class. Destructor calling order: Reverse.
Hierarchical Inheritance
Multiple derived classes from one base class
class Vehicle { // Base class void display() {} }; class Car : public Vehicle { // Derived 1 }; class Bike : public Vehicle { // Derived 2 }; class Bus : public Vehicle { // Derived 3 }; Car c; Bike b; Bus bu; // Each has access to Vehicle functions
Each derived class has its own copy of base class members. Changes in one class don't affect others.
Hybrid Inheritance
Combination of multiple inheritance types
class A { }; class B : public A { }; // Multilevel class C : public A { }; // Hierarchical class D : public B, public C { }; // Multiple // Combination: Multilevel + Hierarchical + Multiple
When using Hybrid Inheritance: Use virtual inheritance to avoid diamond problem if common base exists.
Multipath Inheritance (Diamond)
Multiple paths to same base class
// Without virtual inheritance: PROBLEM class A { int x; }; class B : public A { }; class C : public A { }; class D : public B, public C { }; // A appears twice! // With virtual inheritance: SOLUTION class A { int x; }; class B : virtual public A { }; class C : virtual public A { }; class D : public B, public C { }; // A appears only once!

• virtual keyword prevents member duplication

• Base class initialized only once

• Slightly more memory overhead

• More complex, use only when necessary

Virtual Functions
Runtime polymorphism and dynamic binding
class Animal { virtual void sound() { // Virtual function cout << "Generic sound"; } }; class Dog : public Animal { void sound() override { // Override (C++11) cout << "Bark"; } }; class Cat : public Animal { void sound() override { cout << "Meow"; } }; int main() { Animal *ptr; Dog d; Cat c; ptr = &d; ptr->sound(); // Calls Dog::sound() - Bark ptr = &c; ptr->sound(); // Calls Cat::sound() - Meow }

• virtual keyword enables runtime polymorphism

• Function call resolved at runtime based on object type

• Base class pointer/reference can point to derived object

• Slightly slower than non-virtual (vtable lookup)

• Override keyword (C++11) explicitly marks overriding functions

Virtual Function Rules:

• Must be member function

• Cannot be static

• Must be public or protected

• Same signature in base and derived class

• Virtual in base = automatically virtual in derived

Pure Virtual Functions & Abstract Classes
class Shape { // Abstract class virtual void draw() = 0; // Pure virtual virtual ~Shape() {} }; class Circle : public Shape { void draw() override { // Must implement cout << "Drawing circle"; } }; class Square : public Shape { void draw() override { cout << "Drawing square"; } }; // Shape s; // ERROR: Cannot instantiate abstract class Circle c; // OK c.draw();

• Pure virtual: = 0 in function declaration

• Class with pure virtual = abstract class

• Cannot create objects of abstract class

• Derived class must implement all pure virtual functions

• Used to define interface/contract for derived classes

Friend Functions
Access to private members from outside
class Student { int id; string name; friend void printStudent(Student &s); // Friend declaration }; void printStudent(Student &s) { // Friend function definition cout << s.id << " " << s.name; // Can access private } int main() { Student s; printStudent(s); // Can call non-member function }
Friend Class:
class A { int x; friend class B; // B can access private members of A }; class B { void display(A &a) { cout << a.x; // Can access private } };

• friend grants access to private/protected members

• Not inherited - friend of parent ≠ friend of child

• Breaks encapsulation, use sparingly

• One-way: A declares B friend, not vice versa

• Outside class definition

Key Point: friend violates OOP principle but sometimes necessary. Operator overloading often uses friend functions.
Inline Functions
Request compiler to replace with function body
class Calculator { inline int add(int a, int b) { // Explicit inline return a + b; } int subtract(int a, int b) { // Implicitly inline (inside class) return a - b; } }; inline int multiply(int a, int b) { // Standalone inline return a * b; }

• Compiler replaces function call with function body

• Reduces function call overhead

• Increases code size (code duplication)

• inline is just a request (compiler may ignore)

• Good for small, frequently called functions

• Functions defined inside class = implicitly inline

Use inline when: Function is very simple (1-2 lines) and called frequently. Don't use for complex functions or recursion.

Function Templates
Generic functions for different data types
template // Or template T findMax(T a, T b) { return (a > b) ? a : b; } // Template instantiation cout << findMax(5, 10); // Calls findMax cout << findMax(3.5, 7.2); // Calls findMax cout << findMax("hello", "world"); // Calls findMax
Template with Multiple Parameters:
template void swap(T &a, U &b) { // Swap logic }
Template Specialization:
template // General template void func(T x) { cout << "General"; } template <> // Specialization void func(int x) { cout << "Integer"; }

• Generic function for any data type

• Compiler generates code for each type used

• Template function vs normal function - compiler chooses best match

• Syntax check when instantiated, not declaration

• typename and class keywords are equivalent

Class Templates
Generic classes for any data type
template class Box { T data; public: Box(T val) { data = val; } T getData() { return data; } void setData(T val) { data = val; } }; // Template instantiation Box intBox(10); Box dblBox(3.14); Box strBox("Hello"); cout << intBox.getData(); // 10 cout << dblBox.getData(); // 3.14
Class Template with Non-Type Parameter:
template class Array { T arr[SIZE]; public: void insert(int idx, T val) { arr[idx] = val; } }; Array intArray; Array dblArray;

• Class template definition in header file or same file

• Instantiation syntax: ClassName obj;

• Each instantiation = separate class

• Non-type parameters (int SIZE) must be compile-time constant

• Template class members compiled only when instantiated

Common MCQ Confusion:

• Template function/class = NOT actual code, just blueprint

• Code generated when template is instantiated with specific type

• Each instantiation is separate code
