BIOS (Basic Input/Output System) CRITICAL
▼
What is BIOS?
  • Firmware that initializes hardware before OS loads
  • Runs POST (Power-On Self Test) at startup
  • Provides runtime services for OS and applications
  • Stored in ROM (non-volatile memory)
Key Functions (MCQ Focus):
  • Boot process control - determines boot device order
  • Hardware detection - identifies installed hardware
  • System clock management
  • Device initialization - enables peripherals
  • Security - password protection, boot protection

SMPS (Switched Mode Power Supply) HIGH
▼
What is SMPS?
  • Converts AC power to regulated DC power
  • Uses switching technology (efficient, less heat)
  • Provides multiple voltage rails: +12V, +5V, +3.3V, -12V
  • Contains protection circuits for overcurrent, overvoltage
Advantages over Linear PSU:
  • Higher efficiency (70-90% vs 40-50%)
  • Lighter and smaller
  • Less heat generation
  • Wide input voltage range
  • Faster response to load changes

Processors (CPU) CRITICAL
▼
Key Characteristics to Know:
  • Clock Speed (GHz) - Instructions per second (higher = faster)
  • Cores - Parallel processing units (Dual-core, Quad-core, etc.)

Common MCQ Patterns:

▼

  • Speed: 100 Mbps to 10 Gbps
  • Examples: Office networks, school labs
  • Tech: Ethernet, WiFi
  • Ownership: Single organization

  • Tech: Leased lines, microwave

  • Range: Global (100s of km)
  • Speed: Variable, usually slower
  • Examples: Internet, Satellite networks
  • Tech: Routers, satellites, undersea cables
  • Ownership: Multiple providers

Network Topologies CRITICAL
▼
Bus Topology:
  • Single cable backbone, all nodes tap into it
  • Advantage: Simple, low cost, easy to install
  • Disadvantage: Cable break = network down, difficult to troubleshoot
  • Old Ethernet used this
Ring Topology:
  • Nodes form circle, data travels in one direction
  • Advantage: Organized, no collisions (Token Ring)
  • Disadvantage: One break disrupts entire network
  • Tech: Token Ring (mostly obsolete)
Star Topology:
  • Central hub/switch, all devices connect to center
  • Advantage: One node failure doesn't affect others, easy to manage
  • Disadvantage: Hub failure = entire network down
  • Most common in modern LANs
Mesh Topology:
  • Every device connects to every other device
  • Advantage: No single point of failure, redundant paths
  • Disadvantage: High cost, complex wiring, overhead
  • Full mesh vs Partial mesh (partial is practical)
Hybrid Topology:
  • Combination of two or more topologies
  • Example: Star + Mesh (backbone mesh, star departments)
  • Flexible and scalable

OSI Reference Model CRITICAL
▼
The 7 Layers (Bottom to Top):
  • 1. Physical: Cables, hubs, repeaters, bits
  • 2. Data Link: Frames, MAC addresses, switches, bridges
  • 3. Network: Packets, IP addresses, routers
  • 4. Transport: Segments, TCP/UDP, end-to-end delivery
  • 5. Session: Dialogs, connections, session management
  • 6. Presentation: Encryption, compression, data format
  • 7. Application: HTTP, HTTPS, FTP, SMTP, DNS
★ Memory Trick: "Please Do Not Throw Sausage Pizza Away" (Physical, Data Link, Network, Transport, Session, Presentation, Application)
Key Points for MCQ:
  • Physical layer = raw bits (1s and 0s)
  • Data Link = MAC addresses (addresses within LAN)
  • Network = IP addresses (addresses across networks)
  • Transport = TCP (reliable) vs UDP (fast)
  • Layer 3 devices = Routers; Layer 2 devices = Switches

4-Layer Model (Simpler than OSI):
  • 1. Link Layer: Hardware addressing, PPP, Ethernet
  • 2. Internet Layer: IP, ICMP, routing
  • 3. Transport Layer: TCP, UDP
  • 4. Application Layer: HTTP, HTTPS, FTP, SMTP, DNS, Telnet

LAN Cables & Connectors HIGH
▼
Coaxial Cable:
  • Structure: Inner conductor, insulation, braided shield, outer jacket
  • Impedance: 50Ω (RF) or 75Ω (Cable TV)
  • Old Ethernet: 10Base5 (thick), 10Base2 (thin)
  • Advantages: Good shielding, long distance
  • Disadvantages: Bulky, expensive, obsolete for LANs
  • Connector: BNC connector
Twisted-Pair Cable:
  • Structure: 4 pairs of twisted copper wires
  • UTP (Unshielded): Standard, no shielding, cheaper
  • STP (Shielded): Better EMI protection, expensive
  • Categories: Cat5, Cat5e, Cat6, Cat6a, Cat7, Cat8
  • Standard Ethernet: 10/100/1000 Mbps (most common)
  • Max Length: 100 meters
  • Connector: RJ-45
Optical Fiber Cable:
  • Structure: Glass core, cladding, jacket
  • Signal: Light pulses (immune to EMI)
  • Single Mode: 1 light path, long distance (10+ km)
  • Multi Mode: Multiple light paths, shorter distance (2 km)
  • Advantages: Fastest, longest distance, immune to EMI
  • Disadvantages: Expensive, requires special equipment
  • Connector: SC, ST, LC

Ethernet HIGH
▼
What is Ethernet?
  • Most popular LAN technology
  • Standard: IEEE 802.3
  • Access method: CSMA/CD (Carrier Sense Multiple Access/Collision Detection)
  • Frame format: Destination MAC, Source MAC, Type, Data, CRC
Ethernet Standards:
  • 10Base-T: 10 Mbps over Twisted-Pair
  • 100Base-TX (Fast Ethernet): 100 Mbps
  • 1000Base-T (Gigabit): 1 Gbps (Cat5e+ required)
  • 10GbE: 10 Gbps (Cat6a+ required)
CSMA/CD Process:
  • Carrier Sense: Listen for existing traffic
  • Multiple Access: Share the medium
  • Collision Detection: Detect if collision occurs
  • Backoff: Wait random time, then retry
LAN Devices CRITICAL
▼
Repeater (Layer 1 - Physical):
  • Regenerates signal, extends cable length
  • No intelligence, amplifies noise too
  • Creates single collision domain
Hub (Layer 1 - Physical):
  • Multi-port repeater (4, 8, 16 ports)
  • Broadcasts to all ports (no switching)
  • Single collision domain (all ports share bandwidth)
  • Obsolete - replaced by switches
Bridge (Layer 2 - Data Link):
  • Reads MAC addresses, filters frames
  • Segments collision domains
  • Reduces traffic on each segment
  • Slower than switch (software-based)
Switch (Layer 2 - Data Link):
  • High-speed bridge with many ports
  • Creates separate collision domain per port
  • MAC address learning (learns which MAC on which port)
  • Full-duplex (simultaneous two-way)
  • Standard in modern LANs
Router (Layer 3 - Network):
  • Routes packets between networks using IP addresses
  • Separates broadcast domains
  • Connects LANs together
  • Reads routing tables to forward packets
Gateway (Layer 7 - Application):
  • Converts data formats between incompatible systems
  • Protocol translation (e.g., email gateway)
  • Oldest device type (most complex)
NIC - Network Interface Card:
  • Hardware in each computer
  • Converts data for transmission on network cable
  • Has MAC address (burned-in address)
  • 10/100/1000 Mbps speeds
Modem (Modulator-Demodulator):
  • Converts digital ↔ analog signals
  • Used for dial-up, DSL, cable internet
  • Speed: 56 kbps (dial-up), 50+ Mbps (modern)

Network Protocols HIGH
▼
HTTP (HyperText Transfer Protocol):
  • Port: 80
  • Purpose: Web browsing
  • Method: Stateless, uses TCP
  • Security: No encryption (plain text)
  • Requests: GET, POST, PUT, DELETE, HEAD, OPTIONS
HTTPS (HTTP Secure):
  • Port: 443
  • Purpose: Secure web browsing
  • Security: SSL/TLS encryption
  • Certificate: Uses digital certificates (PKI)
  • Speed: Slightly slower than HTTP due to encryption
FTP (File Transfer Protocol):
  • Port: 21 (control), 20 (data)
  • Purpose: File transfer
  • Method: Two connections (control + data)
  • Security: No encryption (vulnerable)
  • Authentication: Username and password
  • Modern alternative: SFTP (Secure FTP)
SMTP (Simple Mail Transfer Protocol):
  • Port: 25 (standard), 587 (submission), 465 (SMTPS)
  • Purpose: Sending email
  • Method: Push model (client → server)
  • Security: STARTTLS or SMTPS for encryption
  • Pair with: POP3 or IMAP for receiving
Telnet:
  • Port: 23
  • Purpose: Remote login, terminal emulation
  • Security: No encryption (plain text)
  • Replaced by: SSH (Secure Shell)
  • Status: Obsolete due to security concerns

TCP/IP Addressing (IP Classes) CRITICAL
▼
IP Address Format:
  • 32-bit address (IPv4) divided into 4 octets
  • Dotted decimal: 192.168.1.1
  • Each octet: 0-255 (8 bits)
  • Network portion + Host portion (determined by subnet mask)
IP Classes (Classful Addressing):
  • Class A: 1.0.0.0 - 126.0.0.0 | Default mask /8 | 16M+ hosts
  • Class B: 128.0.0.0 - 191.255.0.0 | Default mask /16 | 65K hosts
  • Class C: 192.0.0.0 - 223.255.255.0 | Default mask /24 | 254 hosts
  • Class D: 224.0.0.0 - 239.255.255.255 | Multicast
  • Class E: 240.0.0.0 - 255.255.255.255 | Reserved
Special IP Addresses:
  • 127.0.0.1: Loopback (localhost)
  • 0.0.0.0: Default route, "this network"
  • 255.255.255.255: Broadcast (all hosts)
  • First address: Network address (host bits = 0)
  • Last address: Broadcast address (host bits = 1)
Private IP Ranges (RFC 1918):
  • Class A: 10.0.0.0 - 10.255.255.255
  • Class B: 172.16.0.0 - 172.31.255.255
  • Class C: 192.168.0.0 - 192.168.255.255
  • Not routable on Internet (for local networks)

IP Subnetting HIGH
▼
What is Subnetting?
  • Dividing single network into multiple logical sub-networks
  • Uses subnet mask to determine network/host portions
  • Improves efficiency and security
  • Uses CIDR notation: /24 means first 24 bits = network
Common Subnet Masks:
  • /24 (255.255.255.0): Class C, 254 usable hosts
  • /25 (255.255.255.128): 126 usable hosts
  • /26 (255.255.255.192): 62 usable hosts
  • /27 (255.255.255.224): 30 usable hosts
  • /28 (255.255.255.240): 14 usable hosts
  • /29 (255.255.255.248): 6 usable hosts
  • /30 (255.255.255.252): 2 usable hosts (point-to-point)
Subnet Calculation Formula:
Usable Hosts = 2^(32 - prefix_length) - 2
Example: /24 = 2^(32-24) - 2 = 2^8 - 2 = 256 - 2 = 254 hosts
Example Breakdown (192.168.1.0/24):
  • Network: 192.168.1.0
  • First host: 192.168.1.1
  • Last host: 192.168.1.254
  • Broadcast: 192.168.1.255
  • Usable hosts: 254
★ MCQ Tip: Always subtract 2 from the total (network address and broadcast address can't be assigned to hosts)
