1. Python Basics

Features · Variables · Data types · Indentation · Decision making · Loops

Overview

Key features of Python

  • Interpreted language — executes line by line, no separate compilation step
  • Dynamically typed — no need to declare variable types
  • Object-oriented, functional, and procedural — multi-paradigm
  • Indentation-based block structure (no curly braces)
  • Extensive standard library ("batteries included")
  • Platform independent / portable
  • Open source and free
MCQ tip: Python is interpreted, NOT compiled. Variables don't need type declarations. The creator is Guido van Rossum (1991).

Syntax

Indentation rules

  • Python uses indentation (spaces/tabs) to define code blocks
  • Standard is 4 spaces per level
  • Mixing tabs and spaces causes IndentationError
  • All statements in same block must have same indentation
Common MCQ trap: IndentationError is raised for incorrect indentation. Python does NOT use {} for blocks.
Logic

Control Flow

# if-elif-else
if age >= 18:
    print("Vote")
else:
    print("Wait")

# Ternary operator
status = "Adult" if age >= 18 else "Minor"
MCQ tip: Python has no switch statement in older versions. Python 3.10+ introduced match-case.
Iteration

Loops

for i in range(5):       # 0,1,2,3,4 (5 is exclusive)
    print(i)

while condition:
    pass # placeholder
Important: The else block in a loop runs only if the loop finishes naturally (no break).

2. Data Structures

Lists · Tuples · Sets · Dictionaries

List

Lists []

Ordered, mutable, allows duplicates.

lst = [10, 20, 30]
lst.append(40)      # Add to end
lst.insert(1, 15)   # Add at index
lst.pop()           # Remove last
lst.remove(20)      # Remove by value
MCQ tip: append() adds 1 item, extend() adds multiple items from an iterable.
Tuple

Tuples ()

Ordered, immutable sequence.

t = (1, 2, 3)
# t[0] = 5  # Error! Immutable
MCQ trap: A single element tuple requires a comma: t = (5,). Without the comma, (5) is just an integer.
Set

Sets {}

Unordered, unique elements only.

s = {1, 2, 2, 3} # Result: {1, 2, 3}
s.add(4)
# Access by index is NOT possible
MCQ trap: {} creates an empty Dictionary. Use set() for an empty Set.
Dictionary

Dictionaries {k:v}

d = {"name": "ECET", "year": 2025}
val = d.get("name") # Safe access
MCQ tip: Keys must be immutable (e.g., strings, numbers, tuples). Lists cannot be keys.

3. Object Oriented Programming

Classes · Objects · Constructors · Inheritance

Classes and Methods

class Student:
    def __init__(self, name): # Constructor
        self.name = name
    
    def greet(self):
        print(f"Hi {self.name}")
self represents the instance of the class and is required as the first parameter in instance methods.

Inheritance

  • Single: Child inherits from one Parent.
  • Multiple: Child inherits from multiple Parents.
  • Multilevel: A chain of inheritance (A -> B -> C).
MCQ tip: Use super() to call parent class methods. Python uses MRO (Method Resolution Order).

5. Exception Handling

try · except · else · finally

Handling Errors

try:
    res = 10 / 0
except ZeroDivisionError:
    print("Error!")
finally:
    print("Always runs")
MCQ tip: finally block executes even if a return statement is encountered in try/except.

Threading module · Synchronization
